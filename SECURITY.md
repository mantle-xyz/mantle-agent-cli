# Security Policy

## Reporting a Vulnerability

Do not post vulnerability details in a public issue or pull request.

Preferred path:

1. Use GitHub Security Advisories for this repository if private reporting is enabled.
2. If private reporting is not enabled yet, open a minimal GitHub issue requesting a private security contact.
3. Do not include exploit details, proof-of-concept payloads, private keys, or sensitive endpoint data in that public issue.

## What to Include

When a private reporting path is available, include:

- affected component or file
- impact summary
- reproduction steps
- mitigation ideas, if known

## Scope

This repository includes:

- the local CLI
- the core library
- repository-level skills integration and docs

Security reports should stay focused on this repository and its distributed artifacts.
